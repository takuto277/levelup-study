# 要件定義・設計: コアゲームループ (Core Game Loop)

---

## 1. コンセプト

**「勉強する＝冒険する＝報酬を得る」**

勉強時間とゲーム進行を完全に連動させることで、
「勉強をやめたらキャラも止まる」という設計でモチベーションを維持する。

---

## 2. コアループ図

```
┌─────────────────────────────────────────────────┐
│                                                  │
│   勉強開始（タイマーON）                          │
│       │                                          │
│       ▼                                          │
│   ┌──────────────────────────┐                   │
│   │  勉強中 = 冒険中          │                   │
│   │                          │                   │
│   │  ・パーティが自動でバトル   │                   │
│   │  ・経験値/ゴールド獲得     │                   │
│   │  ・ガチャ石も同時に蓄積    │                   │
│   └──────────────────────────┘                   │
│       │                                          │
│       ▼                                          │
│   勉強終了（タイマーOFF）                          │
│       │                                          │
│       ▼                                          │
│   ┌──────────────────────────┐                   │
│   │  リザルト画面              │                   │
│   │                          │                   │
│   │  ・勉強時間の表示          │                   │
│   │  ・獲得ガチャ石の表示      │                   │
│   │  ・冒険結果（倒した敵/戦利品）│                  │
│   └──────────────────────────┘                   │
│       │                                          │
│       ▼                                          │
│   報酬を使う                                      │
│       ├── ガチャ石 → 召喚（ガチャ）で新キャラ/武器   │
│       ├── ゴールド → 編成画面でキャラ/武器を強化     │
│       └── 経験値 → キャラのレベルアップ             │
│       │                                          │
│       ▼                                          │
│   パーティが強くなる → もっと難しいダンジョンへ      │
│       │                                          │
│       └──→ 「もっと勉強しよう！」 ──→ 最初に戻る   │
│                                                  │
└─────────────────────────────────────────────────┘
```

## 3. 勉強中の報酬設計

### 3.1 ガチャ石（知識の結晶）
| 条件 | 獲得量（案） |
|---|---|
| 勉強10分ごと | 5個 |
| 30分連続達成ボーナス | +10個 |
| 60分連続達成ボーナス | +25個 |
| 1日の累計2時間達成 | +50個 |

### 3.2 冒険報酬（勉強中に自動獲得）
| 報酬種別 | 内容 |
|---|---|
| ゴールド | キャラ/武器の強化に使用 |
| 経験値 | パーティキャラのレベルアップ |
| ドロップアイテム | ダンジョン固有の素材・装備 |

### 3.3 勉強停止時の挙動
- **タイマーを止める → 冒険も即停止**
- 勉強していない間はパーティは「休息中」の表示
- 冒険の途中で止めても、それまでの獲得物は保持される（ペナルティなし）

## 4. タブ構成（確定版）

| 位置 | タブ名 | アイコン | 主な機能 |
|:---:|---|---|---|
| ① 左端 | 冒険 | `map.fill` | 現在のダンジョン進捗、ステージ選択、バトルログ |
| ② 左2 | 編成 | `person.3.fill` | キャラ/武器の装備、強化、レベルアップ |
| ③ 中央 | ホーム | `house.fill` | メインキャラ表示 + 勉強タイマー開始ボタン |
| ④ 右2 | 召喚 | `sparkles` | ガチャ（キャラ/武器の排出） |
| ⑤ 右端 | 記録 | `chart.bar.fill` | 日/週/月の勉強時間グラフ、カレンダー |

### タブの並び順の意図
- **中央（③ホーム）** をアプリの起点と勉強開始の場にする
- **左側（①冒険 ②編成）** はRPG要素（ゲーム寄り）
- **右側（④召喚 ⑤記録）** は報酬と振り返り（成果寄り）
- 左右で「ゲーム」と「勉強」が対称になる美しい構造

## 5. 画面遷移フロー

```
ホーム（③）
  │
  ├── [START] タップ
  │     └── タイマー画面（フルスクリーンモーダル）
  │           ├── 勉強中のカウントダウン表示
  │           ├── キャラがダンジョンで戦っている描写
  │           └── [終了] → リザルト画面 → ホームに戻る
  │
  ├── 冒険タブ（①）
  │     ├── 現在のダンジョン & ステージ表示
  │     ├── 過去のバトルログ
  │     └── 次のダンジョン解放条件
  │
  ├── 編成タブ（②）
  │     ├── パーティ編成（キャラ4体）
  │     ├── 武器装備
  │     └── 強化 / レベルアップ
  │
  ├── 召喚タブ（④）
  │     ├── ガチャバナー表示
  │     ├── 単発 / 10連
  │     └── 排出結果演出
  │
  └── 記録タブ（⑤）
        ├── 日次 / 週次 / 月次の棒グラフ
        ├── カレンダーヒートマップ
        └── 連続日数 / ストリーク表示
```
