# 要件定義・設計: ホーム画面 (Home Screen)

---

## 1. 提案 (Proposal)
- **機能名:** ホーム画面 (Home Dashboard)
- **目的・背景 (Why?):**
  - アプリ起動直後に「自分のこれまでの頑張り（累計時間）」と「所持アイテム」を一目で把握させ、モチベーションを高めるため。
  - 中央の推しキャラクターが動いて出迎えてくれることで、アプリを開くこと自体への抵抗感をなくし、愛着を沸かせるため。
- **概要 (What?):**
  - ユーザーの学習ステータス表示、キャラクター配置、および直感的に「勉強開始（タイマー）」へ移行できるハブ画面。

---

## 2. 要件定義 (Requirements)

### 2.1 ユーザーストーリー
- ユーザーはアプリを開くと、自分が育てた（あるいはお気に入りの）キャラクターのアニメーションを見ることができる。
- ユーザーは画面上部で、これまでの「累計勉強タイム」と「ガチャ用の通貨」を確認できる。
- ユーザーは画面の中央下部にある大きなボタンを押し、迷わず勉強（タイマー画面）を開始できる。

### 2.2 機能要件 (Functional Requirements)
- **ステータス表示機能:** 累計勉強時間（例: `124h 30m`）と通貨の所持数を表示する。
- **キャラクター表示機能:** ユーザーが編成画面で設定した「メインキャラクター」を画面中央にGIFまたはLottie（アニメーション）で表示する。
- **導線（遷移）機能:** 画面下部の大きな「勉強開始」ボタンからタイマー画面へ遷移できること。

---

## 3. 基本設計・技術構成 (Architecture & Design)

### 3.1 画面仕様 (UI/UX)
- **Top Bar (ヘッダー):** 
  - 左上: 累計勉強時間のアイコンと数値表示
  - 右上: ガチャ通貨（仮称：知識の結晶 / マナ / 星座石 など）のアイコンと所持数
- **Center (中央):** 
  - メインキャラクターの待機アニメーション（Idleモーション）。タップすると喋る（吹き出しが出る）などの拡張性を持たせる。
- **Bottom (下部・タブバー):** 
  - 全ての機能にアクセスするためのナビゲーションバー（Tab）を配置します。
  - その最前面（中央）に、大きく目立つ形で「勉強開始（Start Study）」のタイマー起動ボタンを被せて配置（Floating Action Button的デザイン）。
  - **タブ構成（確定版）:**
    1. **⚔️ 冒険 (Quest):** 勉強中に自動進行するダンジョン。バトルログ・ステージ選択
    2. **👥 編成 (Party):** 引いたキャラの武器装備やレベル上げを行う画面
    3. **🏠 ホーム (Home):** 推しキャラの鑑賞＋勉強タイマー開始（中央のメインタブ）
    4. **💎 召喚 (Gacha):** 学習で得た石を使って新しいキャラや武器を引く画面
    5. **📊 記録 (Analytics):** 過去の勉強時間データをカレンダーやグラフで振り返る画面
  - **コアルール: 勉強中 = 冒険中（案A採用）**
    - タイマーON中のみパーティが自動でダンジョンを進む
    - ガチャ石も勉強時間に応じて同時に蓄積される
    - 詳細は `02_CoreGameLoop_Design.md` を参照

### 3.2 データモデル (KMP State)
- **HomeState (ViewModelが吐き出す状態):**
  ```kotlin
  data class HomeState(
      val totalStudyMinutes: Int,
      val currentCurrency: Int, // ガチャ用通貨
      val mainCharacterId: String,
      val mainCharacterImageUrl: String,
      val isLoading: Boolean
  )
  ```
