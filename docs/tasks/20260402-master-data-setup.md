# マスタデータ投入手順（ローカル開発環境）

| 項目 | 値 |
|------|-----|
| 作成日 | 2026-04-02 |
| ステータス | 完了 |
| 担当 | AI / 人間 |

## 概要
ローカル Docker PostgreSQL 環境に、開発・テスト用のマスタデータ（キャラクター・武器・ダンジョン・ガチャバナー）
およびテスト用ユーザーデータを投入する手順を記述する。

---

## 前提条件
- Docker Desktop がインストール済み
- `backend/` ディレクトリで作業する
- `.env` ファイルが設定済み（`make setup` で自動作成）

---

## クイックスタート

```bash
cd backend

# 1. PostgreSQL を起動
make db-up

# 2. Go サーバーを起動（AutoMigrate でテーブル作成）
make run
# → 「✅ マイグレーション完了」が表示されたら Ctrl+C で停止

# 3. シードデータを投入
make seed
```

---

## 投入されるデータ

### マスタデータ（ユーザー共通）

| データ | 件数 | 備考 |
|---|---|---|
| キャラクター (`m_characters`) | 30体 | ★5×5, ★4×10, ★3×15 |
| 武器 (`m_weapons`) | 15本 | ★5×3, ★4×5, ★3×7 |
| ダンジョン (`m_dungeons`) | 6種 | 難易度別（初心者〜伝説） |
| ダンジョンステージ (`m_dungeon_stages`) | 25ステージ | 各ダンジョン3〜5ステージ |
| ガチャバナー (`m_gacha_banners`) | 3種 | キャラ/武器/混合 |

### テストユーザーデータ

| データ | 内容 |
|---|---|
| ユーザー | 2名（「勇者タクト」石1000個 / 「見習い魔法使い」石300個） |
| 所持キャラ | ユーザー1が6体所持 |
| 所持武器 | ユーザー1が3本所持（うち3本装備済み） |
| パーティ編成 | ユーザー1が3スロット編成済み |
| ダンジョン進行 | ユーザー1が「はじまりの森」ステージ3クリア、「水晶の洞窟」ステージ1クリア |

### テストユーザー1の認証情報
- **ユーザーID**: `00000000-0000-0000-0000-000000000001`
- **表示名**: 勇者タクト
- **石**: 1000個
- **ゴールド**: 5000

## Makefile コマンド一覧

| コマンド | 説明 |
|---|---|
| `make db-up` | PostgreSQL を Docker で起動 |
| `make db-down` | PostgreSQL を停止 |
| `make db-reset` | PostgreSQL を停止 + データ削除して再起動 |
| `make seed` | シードデータを投入 |
| `make run` | Go サーバーを起動（AutoMigrate 込み） |
| `make test` | テストを実行 |

---

## データのリセット

```bash
# DB を完全にリセットして再構築
make db-reset
make run    # Ctrl+C で停止
make seed
```

## 画像について

### 開発中
seed.sql では placeholder URL（`https://placehold.co/200x200/色/文字色?text=名前`）を使用している。
アプリ画面には文字入りのカラーブロックが表示される。

### 本物の画像を用意した場合
1. 画像を Cloudflare R2 / Supabase Storage 等にアップロード
2. `m_characters.image_url`, `m_weapons.image_url`, `m_dungeons.image_url` を UPDATE で差し替え
3. または seed.sql の URL を書き換えて `make seed` を再実行

### 推奨画像サイズ
| 用途 | 推奨サイズ | 形式 |
|---|---|---|
| キャラクター立ち絵 | 512×512 px | PNG（透過推奨） |
| 武器アイコン | 256×256 px | PNG（透過推奨） |
| ダンジョンバナー | 800×400 px | JPG or PNG |

---

## 注意事項
- seed.sql は **開発環境専用**。本番環境で実行しないこと
- seed.sql は `TRUNCATE CASCADE` で既存データを全削除してから投入するため、テスト中のデータはすべて消える
- UUID は固定値を使用しているため、複数回実行しても冪等（同じ結果になる）
