## 実装詳細

### 1. 画像の保管場所について
- **開発中**: placeholder URL（`https://placehold.co/200x200/EEE/333?text=CharName`等）を使用
- **本番**: AWS S3 / Cloudflare R2 等のオブジェクトストレージ（無料枠あり）
- **無料で本番も使える候補**: Cloudflare R2（10GB無料）、Supabase Storage（1GB無料）
- seed.sqlの image_url カラムにはplaceholder URLを設定しておき、本物の画像を用意した時点でURLを差し替える運用

### 2. seed.sql の構成
```sql
-- 1. テストユーザー（石1000個、ゴールド5000）
-- 2. キャラクターマスタ（30体: 星5×5体、星4×10体、星3×15体）
-- 3. 武器マスタ（15本: 星5×3本、星4×5本、星3×7本）
-- 4. ダンジョンマスタ（6種）+ ステージ（各3-10ステージ）
-- 5. ガチャバナー（3種）+ 排出テーブル（JSON）
-- 6. テストユーザーの所持キャラ（6体）
-- 7. テストユーザーの所持武器（3本）
-- 8. テストユーザーのパーティ編成（3スロット）
-- 9. テストユーザーのダンジョン進行状況
```

### 3. ダンジョン設計
- ダンジョンのゴール＝ポモドーロ時間に連動（推奨勉強時間で最終ステージ到達）
- 勉強時間の進捗バーとダンジョン進行がリンク
- 各ステージの enemy_composition はJSONB形式で敵名・HP・ATKを定義

## リスク・注意点
- seed.sql は開発用のみ。本番環境では管理画面から投入する想定
- UUID は固定値を使用（テスト間の再現性のため）
- 画像URLはplaceholderのため、実際の見た目は仮
